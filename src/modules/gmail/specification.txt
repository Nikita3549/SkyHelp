------------ ENGLISH ------------
At the core of this module is a composition of submodules.
Gmail module – the main module that consolidates the logic for handling emails related to Gmail.
Specifically, the Gmail service contains the logic for interacting with the API via google-apis.
The service itself is made up of reusable methods within its submodules, such as sending an email, reading it, and so on.
The Gmail service includes main entities (submodules) such as:
attachment – handles working with email attachments outside the Gmail API level,
meaning everything related to file storage and database records;
email – works with the email entity from the database;
and an accounts folder containing different accounts for various purposes, such as @office and @noreply.
Each entity is implemented as a separate module.

------------ RUSSIAN ------------
В основе этого модуля лежит композиция из подмодулей

Gmail module - основной модуль который собирает в себя логику работы с письмами связанными с gmail,
конкретно в gmail service лежит логика общения с api посредством google-apis.
Сам сервис состоит из переиспользуемых методов в подмодулях, по типу отправка письма, его чтение и тд.

Gmail service содержит основные сущности(подмодули) такие как:
attachment - работа с приложениями к письмам вне уровня gmail api,
т.е все что связано с хранением файлов и записями в бд, email - работа с сущностью email
из базы данных, а также в папке accounts разные аккаунты для разных целей по типу @office и @noreply.
Каждая сущность - отдельный модуль.
